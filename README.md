# generative-reading · 生产型阅读

用「生产型阅读」的方式吸收一篇内容：不是读它讲了什么，而是反向拆解作者的创作决策，把可复用的手艺萃取出来并沉淀进自己的知识网络。

## 这个 skill 做什么

- **浏览**（浅）：快速扫大量内容，捡和已知不一样的信息，铺判断力基线
- **对标**（中）：被某个点卡住后往外查证延伸，把一件事搞明白，长世界认知
- **精读**（深）：逆向拆解作者的创作决策，挖出不可见的选择，长写作手艺
- **复利系统**：强迫输出 → 生长优先 → 还卡 → 两棵树 → 传动轴正反馈

## 触发词

「帮我读/拆解/分析这篇」「学习借鉴」「提升写作」「搞懂它为什么好」「我能学到什么」「复习收获库」「每天看新增卡片」「这篇值不值得细读」

## 三层分工

| 层级 | 位置 | 内容 | 加载时机 | 可替换性 |
|------|------|------|----------|----------|
| 世界观 | `SKILL.md` | 怎么想：判断引擎、四笔账、三族方法 | 始终加载元数据，触发后加载正文 | 不可替换 |
| 方法 | `references/` | 怎么做：每个档位的具体操作流程 | 进入对应档位时按需读取 | 可替换 |
| 积累 | `memory/` | 读过什么：收获卡、两棵树、索引 | 每次阅读前回查，阅读后更新 | 越用越厚 |

## 设计哲学

参照一泽 Eze 三段式：策略哲学 + 最小完备工具集 + 必要的事实说明。核心是避免过度指定「该怎么做」，转而校准「怎么想」。换一个任务还成立的是哲学（进 SKILL.md），一换情况就只能照搬的是做法（沉到 references/）。

一句话总纲：**注意力最稀缺，先用最便宜的方式找出哪里不一样，再把最贵的注意力留给反推作者是怎么想的。**
