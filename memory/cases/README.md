# 案例卡·字段与还卡规范（迁移自旧库头部）

# 收获库（双向 · 多维度 · 可生长 · 会还卡）

> 这是这台引擎的**传动轴实体**。它不是存档柜，而是阅读回路的一部分：
> - **双向、读取两用**：写入端＝每次阅读的收获沉淀于此；读取端有两种：**被动回查**（被眼前这篇触发，按母题/领域捞旧积累，用过往认知发动这次的勾连与深读；读到"车企倒闭"先来捞"汽车行业/商业"下的旧积累）与**主动还卡**（见下）。
> - **会还卡（把库从"参考资料"变成"老师"的关键）**：每次产出当日收获时，库不只增新卡，还**主动还回几张值得重看的旧卡**（按勾连现判，宁少而真），优先还**能和今天这篇勾连上的**、以及**攒了很久却从没真的调用过的**。目的是巩固：沉淀进库 ≠ 学会了，学会要在新语境里重遇、被再次调用。这样"每天看新卡＝完成学习"才成立：当日学习＝今天的新卡 ＋ 今天值得重遇的旧卡。**按勾连度和久未调用来还，别用"每 N 天复习"的死间隔。**
> - **多维度、厚度随档**：按性质分五类收：**信息/行业认知、心智模型/认知角度、手艺/做法、好表达、坏表达（避坑）**。**卡的厚度跟着处理档走**：浏览档产出轻信息卡，精读档才产出教材级手艺卡；别强求每张都教材级，那是在卡片层面又废掉减法。不是每篇都到得了"手艺"，但**永不为零**：再轻的内容至少带走一条信息/常识。
> - **可生长（生长＝勾连的一种结果）**：条目**只融合、不重复**。读到同一认知（如"以小见大"）的新细节/新适用条件，**回到原条目补全**，不新建。写入逻辑＝先查同主题条目：有则 merge 补全/修正，无则新建。这最常发生在**对标档的勾连里**：为读今天这篇去回查老卡，正好把今天的新料并进那张老卡。
> - **自动且可见**：写入与还卡每次默认自动做，并**明确告诉用户**：这次带走了什么、归哪一类、新建还是补全了哪条、今天还回了哪几张旧卡及为什么还它。

## 条目字段（统一格式，支持生长）

> 一条收获必须**自带头尾**：脱离当时那次对话、隔很久回看，它仍能自我解释、能被验证、能被复用。所以四栏缺一不可，它们正是这条收获将来"被回查、被还卡时能被调用"的接口：**出处**让你能验证、能联想到相似场景；**怎么用**让你能落地。**严禁"来源：本篇"这类不落地的偷懒写法。**
>
> **精读卡的额外门槛，教材级：** 精读档产出的手艺卡，标准不是"未来的我能凭它找回原文"，而是**没看过原文的人也能从它学会这个动作**。所以"出处与还原"要**重演到动作可见**：把作者那一手具体做了什么还原出来，而不只是命名它（"用场景开头"是命名、不够，要让人看见他怎么用）。但**重演 ≠ 复刻原文**：传递创作决策和它为什么 work，配极少量示意性引用即可，绝不大段搬原文进卡。轻信息卡不必到这个门槛。
>
> **好表达/坏表达卡的额外门槛，重演到修辞动作：** 表达类卡下钻在句以下，门槛是**把作者的具体语言动作还原到可复演**，光命名一律不够（"用了比喻""场景开头"是命名，不算）。一张达标的表达卡至少含三样：①那个**具体词句**（出彩的神来之笔，或功能标尺扫出的语病/啰嗦/夹私货处）；②一次**替换/删除/语序试错**：把它换成最普通的说法，写出**丢了什么**（这既是"为什么 work"的证据，也是它和命名的分水岭）；③它**跨过了什么**：移用了哪个语域 / 用了反直觉喻体 / 在全篇是否复现承重。同样**重演 ≠ 复刻**：引用压到示意性的短句，传的是"换一下就塌"的那个机制，不是原文。

```
### [类别] 认知内核（一句话：它是什么）
- 出处与还原：从哪篇的哪个具体地方来（原话/场景/数据），以及**作者在那里到底做了什么处理**，精读卡要**重演到动作可见、没看过原文的人也能学会**（不只命名，但不复刻原文）；轻信息卡还原到能验证、能联想到相似场景即可
- 为什么成立：背后的隐性逻辑/动因（最重要的一栏，是这条能迁移的根据）
- 怎么用：何时该想起它调用 + 一句**可直接照做**的复用提示 + 容易翻车在哪
- 关联：→ 相关卡名（关系说明）。标出这张卡和库里哪些卡有连接："解释了""被XX解释""同一次延伸串起""互为正反面""同属XX趋势"。对标档尤其要填：搞清楚一件事时勾连的几个点之间的线路，下次回查拉一张就能顺藤摸到一串。浏览档的轻信息卡如果没有明确关联可不填。
- 来源与更新：贡献过它的篇目（逐次累加）｜最近更新 YYYY-MM-DD
```

> 生长时：读到同主题的新例子/新适用条件，回原条目：在"出处与还原"里**追加新案例**、在"为什么成立/怎么用"里**补全或修正**，在"关联"里检查相关卡是否也需要同步更新，并在"来源与更新"累加篇目、刷新日期。

## 还卡怎么做（巩固机制，不是死间隔算法）

> 每次产出当日收获时，除新卡外，主动还回**几张**旧卡（按勾连现判，宁少而真）。选卡按下面的优先级（性向，不是公式）：
> 1. **能和今天这篇勾连上的**：今天读的东西激活了哪张老认知？把它还出来，让新旧互相印证、顺手判断要不要"生长"补全它。这是最高优先级，因为带语境的重遇最长记。
> 2. **久未调用的**："来源与更新"里日期很旧、且从没在后续阅读里被回查/复用过的好卡。攒了不用＝白攒，主动捞出来重看一遍。
> 3. 信息熵高的优先于轻信息卡：手艺卡、心智模型卡值得反复重遇；一条孤立的信息/常识还一次即可，不必反复。
>
> **呈现方式**：还卡时一句话说清"为什么今天还它"（"今天这篇的留白手法，和你上次存的《一只鸟》'留白当引擎'是同一招，可对照"），而不是默默把旧卡贴出来。让用户每天的"看新卡＋重遇旧卡"都带着勾连线，而非两堆无关的卡。
> **别做的事**：不要设"每 3 天/每 7 天复习"这种固定排程；不要每天把整库翻出来；不要为凑数还无关的卡。没有值得还的就不还，宁缺毋滥。
