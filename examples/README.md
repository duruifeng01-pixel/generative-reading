# 范例（身份：证据，不是模板）

这里的文件展示引擎显形之后长什么样、合格线在哪里。使用规则：

- 范例是证据，照抄结构等于没学。每份好坏对照的批注里都标了“特有处”：那些是该篇稿子独有的运气与语境，换一篇必须重新做一遍反推，不许搬结论。
- 弱模型对照的是坏版批注（先认出自己常犯的形态），强模型对照的是好版的深度（这是下限不是上限）。

清单：
- `gold-archive-shaoyihui.md`：金标准拆解档案（《邵艺辉 轻盈一跃》），语言与判断的首选锚。注意它怎么解释一个手法、怎么过渡、怎么下判断、敢用多日常的话；正文里没有引擎和库的黑话。写档案前先重读它校准语感。
- `gold-archive-yangzhichang.md`：金标准拆解档案（《大哥在养殖场看着你》），档案语气、引文随行、好坏都讲的整体参照。
- `exemplar-title-drill.md`：标题级深拆的好坏对照，演示生产型回路的合格线与最常见的假执行形态。
- `exemplar-expressiveness.md`：表现力组合分析的好坏对照，演示拆到组合层的合格线。
